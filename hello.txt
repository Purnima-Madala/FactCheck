## PROJECT OVERVIEW

NeuralArena is a multi-AI wrapper and evaluation platform. The user types 
a single prompt, sends it to multiple AI models simultaneously, and sees 
a side-by-side comparison of their responses with computed quality metrics:
- Hallucination Rate (factual accuracy scoring)
- Correctness Score (answer quality)
- Relevance Score (how on-topic the response is)
- Confidence Score (model self-certainty heuristic)
- Response Latency (time to first token / full response)
- Token Efficiency (answer quality per token used)

- Parallel async calls to multiple AI provider APIs using Promise.allSettled
- A scoring engine module that evaluates responses against each other
  and against a reference answer (if provided)
- Rate limiting middleware (express-rate-limit)
- CORS configured for local dev
- dotenv for API key management

AI Providers to integrate (each behind an abstraction layer):
- OpenAI (GPT-4o)
- Anthropic (Claude Sonnet)
- Google Gemini (1.5 Pro)
- Mistral (mistral-large)
- Cohere (Command R+)

Each provider must be wrapped in a standardized adapter interface:
  { modelId, name, fetchResponse(prompt): Promise<ModelResponse> }
where ModelResponse = { text, latencyMs, tokensUsed, rawMetadata }

## API CONTRACT

POST /api/arena/evaluate
Request body: {
  prompt: string,
  referenceAnswer?: string,   // optional ground truth
  models: string[]            // which model IDs to query
}

Response: {
  results: [
    {
      modelId: string,
      name: string,
      response: string,
      metrics: {
        hallucinationRate: number,   // 0-100 (lower is better)
        correctnessScore: number,    // 0-100
        relevanceScore: number,      // 0-100
        confidenceScore: number,     // 0-100
        latencyMs: number,
        tokensUsed: number
      },
      compositeScore: number         // weighted average
    }
  ],
  winner: string,                    // modelId with highest compositeScore
  evaluatedAt: string                // ISO timestamp
}

## SCORING ENGINE LOGIC

scoringEngine.js must implement:

1. Relevance Score: Use TF-IDF cosine similarity between prompt and 
   response. No external NLP library — implement from scratch.

2. Hallucination Rate: Cross-reference all model responses. 
   Sentences that appear in only one model's response and contradict 
   the majority are flagged. Score = % of flagged sentences.

3. Correctness Score: If referenceAnswer is provided, compute overlap 
   similarity with response. If not, use inter-model consensus scoring 
   (responses that agree with the majority get higher scores).

4. Confidence Score: Heuristic — detect hedging phrases ("I think", 
   "might be", "I'm not sure", "possibly") and reduce score accordingly. 
   Also penalize for very short responses.

5. Composite Score = 
   (correctness * 0.35) + (relevance * 0.25) + 
   ((100 - hallucination) * 0.25) + (confidence * 0.15)

## 3D VISUAL DESIGN

ArenaScene.jsx:
- Dark space-like background (#050510)
- One glowing 3D orb per AI model, arranged in a circular orbit
- Each orb has a unique color (OpenAI=cyan, Anthropic=violet, 
  Gemini=amber, Mistral=emerald, Cohere=rose)
- On query submission: orbs animate inward toward center (battle start)
- After results arrive: orbs animate to a leaderboard formation 
  (winner moves to top/center, larger, brighter)
- Orb glow intensity = compositeScore (higher score = brighter orb)
- Add particle trails behind orbs when animating
- Use drei's <Float> and <Sparkles> components for ambient motion
